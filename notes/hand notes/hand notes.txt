1. System Overview

The hand system manages the player's cards during battle.
Cards flow in one direction:

global.player_collection
        ↓
global.battle_deck  (saved by deckbuilder on ready)
        ↓
OBJ_Deck  (loads and shuffles battle_deck on room start)
        ↓
OBJ_Hand  (draws cards from OBJ_Deck into hand[])


OBJ_Deck stores card IDs only
OBJ_Hand stores full card data structs
global.player_collection.owned never changes during battle

3. Script Reference


SCR_Hand_Init


Entry point for OBJ_Hand Create event




Calls SCR_Hand_Create() to initialize variables
Draws 5 opening cards from OBJ_Deck
Change the number 5 to adjust opening hand size


Depends on: SCR_Hand_Create, SCR_Hand_DrawFromDeck

SCR_Hand_DrawFromDeck


Draws one card from OBJ_Deck into hand




Uses 'with' scope to call OBJ_Deck functions correctly
Returns true on success
Returns false if: deck empty, hand full, OBJ_Deck not found


Call this whenever the player needs to draw a card (start of turn, button press, etc.)

Depends on: OBJ_Deck, deck_DrawCard, deck_GetCardData, hand_AddCard

SCR_Hand_GetSpacing()


Calculates pixel spacing between cards based on hand size




6 or fewer cards → normal spacing (85px)
More than 6 cards → compresses to fit all cards in the same width
Never compresses below minimum spacing (30px)


Returns: spacing value (real number)


SCR_Hand_IsHovered(_index, _spacing)


Returns true if the mouse is over the card at the given index




Accounts for overlapping cards
If mouse is past the start of the next card, current card is NOT hovered
Only one card can be hovered at a time

SCR_Hand_Draw


Main draw function — call in OBJ_Hand Draw event



Runs two passes:


Pass 1: draws all non-hovered cards
Pass 2: draws hovered card on top of everything


The two-pass approach ensures the hovered card always appears above overlapping cards.

Depends on: SCR_Hand_GetSpacing, SCR_Hand_IsHovered, SCR_Hand_DrawCard

5. Important Rules


hand[] stores full card DATA structs (id, name, type, level, etc) — not IDs
OBJ_Deck stores card IDs only — SCR_Hand_DrawFromDeck converts them to structs
global.player_collection[].owned NEVER changes during battle
hand_Count is the source of truth for how many cards are in hand
OBJ_Deck must be higher than OBJ_Hand in Room_battle creation order
OBJ_GameController must be first in creation order so card_DB exists
